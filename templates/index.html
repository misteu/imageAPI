Image-Cropping-Microservice

This WebApp (in fact, the micro-service behind it) crops an image posted via POST-Request and responds the cropped image as jpeg.
It is powered by python with the help of some libraries (see FAQ below)

The cropping is defined via pixel-coordinates of the upper-left and lower-right corner of a box. Therefore, the pixel-value for Cropping-Left has to be smaller than Cropping-Right and Cropping-Top has to be smaller than Cropping-Lower. Coordinates can be negative and also greater than the orginal image's dimensions if you followed the rules before. The service's response is a pure image/jpeg that can be viewed in your browser, saved or processed by anything calling the API.

FAQ:

How to use the API-endpoint of Image-Cropping-Microservice

The actual microservice is located at http://leachim.pythonanywhere.com/cropper and can be requested by any HTTP-Client (e.g. the WebApp above) via the POST-method. For a valid request there are some rules to follow:

Parameter Argument
API-endpoint HTTP-POST @ http://leachim.pythonanywhere.com/cropper
content-type multipart/form-data
file an image file that can be handled by Pillow. Tested with .PNG and .JPEG/.JPG, .GIF-files do not work.
left some Integer smaller than right
upper some Integer smaller than lower
right some Integer greater than left
lower some Integer greater than upper

The parameters and arguments can be included in form-data or as URL-arguments (see example in error-JSON below). You have to give all the arguments in one of these two ways, otherwise you will get an error-JSON.

Error-Responses

Errors are handled with JSON-Responses in a format like this: 

{
"#error": "Invalid dimensions for area to crop. Requirements: upper < lower und left < right",
"exampleURL": "http://leachim.pythonanywhere.com/cropper?left=100&upper=200&right=300&lower=400"
}

On pythonanywhere.com all the network latency and other limitations are added, leading to around 600ms for the smaller image and around 46s for the big image (with my internet-connection from Germany). Anyway, it fulfills its purpose for being a showcase.

Can I have a link to my cropped image?

No you can't, because there is nothing saved to the server's filesystem. My goal was to create an API that manipulates an image without actually saving it. The request's body content is handed directly to Pillow and Pillow's cropped image is responded via Flask to the client. There is no file.save() executed inbetween, in fact the cropped image is only generated for that specific response. Maybe, it is cached by pythonanywhere.com's servers somehow but not by this API's backend.
